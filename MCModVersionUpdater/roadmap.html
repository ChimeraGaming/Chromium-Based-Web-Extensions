Roadmap

Last updated: March 25, 2026 (v1.0.0)

What We Just Shipped

  • Upgrade mode is now first-class, and Downgrade mode is fully wired in the same flow.
  • Roadmap and Settings open consistently from popup and injected panel.
  • Settings were cleaned up and simplified to only the controls that matter.
  • Duplicate handling and UI consistency got another pass for cleaner scan sessions.
  • The results view now supports theme switching: Light, Dark, Github (Dark), and SNES.
  • Results now include a mode-aware column: Upgradable in Upgrade mode, Downgradable in Downgrade mode.
  • Export is now focused on JDownloader2: `.meta4` output from found direct links, scoped to active filters, with a custom filename prompt before save.
  • Folder scanning was reworked to avoid the 64MiB extension message limit by sending lightweight metadata instead of full payloads.
  • API handling is now settings-first, with CurseForge and Modrinth keys managed directly in the Settings page.
  • When CurseForge API is missing, scan output now says ADD CURSEFORGE API instead of Unknown.
  • Match links are now split into Direct - Source and Site - Source so direct download/export paths stay clean.
  • Results now include Last Updated controls with date filtering and Newest/Oldest sorting.
  • Header text filters now support multi-token input with semicolons (for example: a;b;c).
  • A new optional fuzzy replacement search can use mod descriptions to suggest better candidates on major version jumps.
  • When ignoring already-converted mods, panel flow now supports required two-pack comparison (Old + New) before running update search.
  • Advanced toggles and theme control were moved to Settings so the in-page panel stays focused on scan flow.

Current Focus

  • Improve version recommendation confidence when mods have renamed files or unusual packaging.
  • Keep reducing UI friction so scan-to-export stays fast, especially on large packs.
  • Tighten source clarity when multiple providers or partial matches exist.

Later Upgrades

  • Deeper dependency awareness (optional warning pass before export).
  • Manual match correction for edge cases where automatic matching is close but not perfect.
  • Optional pack rebuild flow after review and link validation.
  • Richer source integrations for curated external repositories.

Flow We Are Optimizing

Upload pack or folder -> choose Upgrade or Downgrade -> set target version -> scan -> review mode column and matches -> export `.meta4` for JDownloader2.
